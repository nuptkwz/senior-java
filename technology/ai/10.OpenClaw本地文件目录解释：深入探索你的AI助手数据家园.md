大家好，我是你们的技术老友**科威舟**，今天给大家分享一下OpenClaw本地文件目录。

> 当你第一次成功部署OpenClaw后，打开用户主目录下的`.openclaw`文件夹，可能会被眼前的一堆文件和目录搞得眼花缭乱。别担心，这其实是你的AI助手精心构建的"数字家园"！今天，我们就来彻底解密这个神秘目录的每一个角落，让你从文件系统的角度真正理解OpenClaw是如何工作的。

## 为什么需要了解这些目录？

OpenClaw采用**本地优先**的设计哲学，所有数据默认存储在本地文件系统中。这意味着：

- **完全的数据控制权**：你的对话历史、记忆、配置都在自己设备上
- **无数据库依赖**：使用纯文本文件存储，易于备份和迁移
- **透明可审计**：每个文件都有明确用途，随时可以查看和编辑
- **隐私保护**：敏感信息不会自动上传到云端

了解这些目录结构，不仅能帮助你更好地管理和维护OpenClaw，还能在出现问题时快速定位和解决。

## 核心目录结构总览

OpenClaw在用户主目录下创建`~/.openclaw/`目录（旧版本可能为`~/openclaw`），这是整个系统的**核心状态目录**。让我们通过表格来直观了解每个部分的作用：

### 根目录文件与目录

| 文件/目录 | 类型 | 核心功能说明 |
|-----------|------|--------------|
| `openclaw.json` | 配置文件 | **主配置文件**，包含Agent、Provider、全局工具策略等所有核心配置，使用JSON5格式（支持注释） |
| `openclaw.json.bak` / `.bak.1` | 备份文件 | 配置自动备份，用于配置回滚，防止误操作导致配置丢失 |
| `update-check.json` | 状态文件 | 版本更新检查记录，记录最近检查更新的时间和结果 |
| `exec-approvals.json` | 安全文件 | 执行审批记录，记录已批准/拒绝的危险操作，增强安全性 |
| `agents/` | 目录 | **Agent配置目录**，每个子目录对应一个独立Agent，包含会话历史、个性化设置等 |
| `credentials/` | 目录 | **凭证存储目录**，API密钥、OAuth令牌等敏感信息的安全存储位置 |
| `extensions/` | 目录 | **扩展插件目录**，通道插件、功能扩展等第三方或自定义插件 |
| `skills/` | 目录 | **技能包目录**，从ClawHub安装的技能（如weather、github等） |
| `workspace/` | 目录 | **Agent工作区**，Agent唯一可操作的工作目录，包含记忆、规则等核心文件 |
| `logs/` | 目录 | **日志目录**，运行日志和调试信息，便于问题排查 |
| `browser/` | 目录 | 浏览器自动化相关数据，CDP会话、缓存等浏览器控制所需文件 |
| `canvas/` | 目录 | 可视化工作区数据，实时Canvas界面的状态存储 |
| `cron/` | 目录 | 定时任务配置和状态，Cron作业的定义和执行记录 |
| `devices/` | 目录 | 已连接设备的管理信息，跨设备控制所需的配对和设备信息 |
| `identity/` | 目录 | 身份认证相关数据，用户身份和权限管理信息 |
| `completions/` | 目录 | 自动补全历史或缓存，提升交互效率的辅助数据 |

### workspace/ 目录详解

workspace是OpenClaw的"大脑中枢"，Agent在这里"生活"和工作。以下是workspace目录下的关键文件：

| 文件/目录 | 作用 | 必填 | 说明 |
|-----------|------|------|------|
| `AGENTS.md` | 助手行为准则与工作规范 | ✅ | 定义助手在每个会话中的行为规范、安全边界和工作流程，相当于"员工手册" |
| `SOUL.md` | 助手的核心人格与价值观 | ✅ | 定义助手的人格特质、价值观和响应风格，塑造独特的"个性" |
| `IDENTITY.md` | 助手的身份定义 | ✅ | 明确助手的身份信息，如名称、角色、能力范围等 |
| `USER.md` | 用户信息与偏好 | ✅ | 记录用户的个人信息、偏好、习惯等，使助手能提供个性化服务 |
| `TOOLS.md` | 本地工具与环境配置 | ⭕ | 定义本地工具的使用方式和环境配置，非必需但建议配置 |
| `MEMORY.md` | 长期记忆存储 | ⭕ | **精选记忆**文件，存放经过筛选和提炼的长期记忆精华，仅在私聊时加载 |
| `BOOTSTRAP.md` | 首次启动引导 | ⭕ | 首次运行的"出生证明"，记录初始化过程和基础设置 |
| `HEARTBEAT.md` | 周期性任务清单 | ⭕ | 心跳检查清单和提醒事项，定义定期执行的任务 |
| `skills/` | 技能扩展目录 | ⭕ | 技能文件目录，存放从ClawHub安装或自定义的技能 |
| `memory/` | 每日记忆日志目录 | ⭕ | **每日记忆**存储，按日期组织的原始流水账记录，采用Append-only（只追加）模式 |

### agents/ 目录结构

每个Agent在`agents/`目录下都有独立的子目录，以Agent ID命名。以主Agent（通常为`main`）为例：

| 文件/目录 | 作用 | 说明 |
|-----------|------|------|
| `sessions/` | 会话存储目录 | 存放所有会话的历史记录和元数据 |
| `sessions/sessions.json` | 会话元数据 | 存储会话元数据，如sessionId、token计数、memoryFlush状态等 |
| `sessions/<sessionId>.jsonl` | 详细会话历史 | 以仅追加（Append-only）的JSONL格式记录完整的对话树，包括消息、工具调用和压缩摘要 |
| `auth-profiles.json` | API认证配置 | Agent特定的API认证配置信息 |
| `models.json` | 模型配置 | Agent使用的模型配置，可覆盖全局设置 |
| `qmd/` | QMD后端数据 | 用于存放QMD后端的索引与配置（如果启用） |

## 关键文件深度解析

### 1. openclaw.json - 系统的心脏

这是OpenClaw最重要的配置文件，采用JSON5格式（支持注释和`$include`指令）。主要包含以下部分：

- **meta**：元数据，记录最后修改版本和时间戳
- **gateway**：网关配置，包括端口、运行模式、绑定地址、认证等
- **agents**：代理配置，定义默认模型、工作空间、并发限制等
- **models**：模型配置，支持多种AI模型提供商
- **channels**：频道配置，定义各消息平台的接入设置
- **tools**：工具配置，启用和配置各种功能工具

### 2. MEMORY.md 与 memory/ - 记忆系统的双引擎

OpenClaw的记忆系统是其核心亮点，采用**可读、可编辑的Markdown文件**承载整个记忆系统：

- **MEMORY.md**：类比人类的长期记忆，包含经过筛选和提炼的精华。仅在主会话（与用户一对一聊天）时加载，不会在群聊或共享会话中暴露。
- **memory/YYYY-MM-DD.md**：每日的原始流水账记录，存储在workspace的memory/目录下。采用混合模式：既包含原始交互日志，也包含Memory Flush产生的自动摘要。系统在每次会话时自动加载"今天+昨天"的内容。

### 3. 会话管理机制

OpenClaw的会话管理非常精细：
- **主会话**：每个智能体的一个直接聊天会话，键为`agent:<agentId>:<mainKey>`（默认main）
- **会话分组**：通过`session.dmScope`控制私信如何分组，支持多种隔离策略
- **状态存储**：所有会话状态都由Gateway网关拥有，UI客户端必须向Gateway查询会话信息

## 实用技巧与最佳实践

### 1. 备份策略
- 定期备份整个`~/.openclaw/`目录
- 重点关注`openclaw.json`和`workspace/`目录
- 利用自动生成的`.bak`备份文件进行快速回滚

### 2. 故障排查
```bash
# 检查Gateway状态
openclaw gateway status

# 运行健康检查
openclaw doctor

# 查看日志
tail -f ~/.openclaw/logs/openclaw-*.log
```

### 3. 性能优化
- 定期清理`logs/`目录中的旧日志文件
- 监控`browser/`目录大小，避免浏览器缓存占用过多空间
- 对于长期运行的Agent，考虑定期归档`sessions/`中的历史数据

### 4. 安全建议
- 保护`credentials/`目录的访问权限
- 定期审查`exec-approvals.json`中的审批记录
- 谨慎配置`agents/defaults`中的权限设置

## 环境变量覆盖

OpenClaw支持通过环境变量覆盖默认路径：
- `OPENCLAW_HOME`：设置主目录路径
- `OPENCLAW_STATE_DIR`：覆盖状态目录位置
- `OPENCLAW_CONFIG_PATH`：覆盖配置文件路径
